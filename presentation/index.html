What the Agent Discovered

Root causes ranked by impact, identified autonomously

PRIMARY — Wrong NIC selected after RHEL 9.7 migration

System had a 100 Gbps NIC (ens2f0np0) but traffic was routed through a 25 Gbps NIC (eno12399np0). A 4× bandwidth bottleneck hiding in plain sight.

SECONDARY — Nginx misconfigured for high-RPS workloads

worker_connections 1024 · access_log on at 380K+ RPS · no file caching · no async I/O

TERTIARY — Kernel TCP stack at defaults

208 KB network buffers · cubic congestion control · somaxconn 4096

Key insight: The NIC bug alone delivered 3–4× more improvement than all other optimizations combined. Classic migration regression — invisible to traditional monitoring.
